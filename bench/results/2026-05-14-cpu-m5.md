# 2026-05-14 — Apple M5 CPU run

Same methodology as the [2026-05-09 M1 run](./2026-05-09-cpu-m1.md);
new hardware. This file is the 50 RPS single-point comparison. The
matching sweep at 10/50/100 RPS is in
[`2026-05-14-sweep-cpu-m5.md`](./2026-05-14-sweep-cpu-m5.md).

## Setup

| Field | Value |
|---|---|
| Hardware | Apple M5 (10 cores), macOS 26.4 |
| Python | 3.12.13 |
| Model | `amazon/chronos-bolt-tiny` (~80 MB) |
| Workload | 64-step history → 12-step horizon, `output_mode=mean` |
| Target RPS | 50 (sustained, open-loop) |
| Warmup | 5 s |
| Measurement window | 20 s (~1000 samples) |
| Batch params | `max_batch_size=8`, `batch_wait=10ms` (all three) |
| Replicas / workers | 1 / 1 (all three) |
| sheaf-serve | v0.10.0 |
| Ray | 2.55.x |
| BentoML | 1.4.39 |

## Latency (steady-state, post-warmup)

| Framework | p50 (ms) | p95 (ms) | p99 (ms) | Mean (ms) | Max (ms) | Errors |
|---|---:|---:|---:|---:|---:|---:|
| **sheaf-serve** | 20.5 | 22.7 | 24.7 | 20.6 | 42.8 | 0 |
| Raw Ray Serve | 19.2 | 21.0 | 22.1 | 19.4 | 45.9 | 0 |
| BentoML | 5.6 | 6.8 | 7.3 | 5.6 | 13.0 | 0 |

All three sustained 50.05 RPS over 20 seconds (target 50.00; the
0.1% overshoot is the open-loop tail from the warmup boundary).

## What this shows

**sheaf vs raw Ray Serve — parity, with a tight tail.** sheaf is 1.3 ms
slower at p50 (20.5 vs 19.2 ms) and 0.6 ms slower at p99 — the typed-
contract layer adds a small per-request cost that is real but not
material at 50 RPS. The whole sheaf hot path is "Ray Serve + Pydantic
validation"; the gap matches that overhead.

**sheaf vs BentoML — BentoML wins single-replica latency.** BentoML's
p50/p95/p99 are roughly a quarter of sheaf's. The cause is the same as
on M1: Ray Serve's multi-actor request routing (`client → HTTP proxy
actor → replica actor → batched call → reply path`) pays per-hop
overhead per request, where BentoML's in-process adaptive batching does
not. The trade-off is intentional: Ray Serve gives you horizontal scale-
out, hot-swap, and multi-replica orchestration in exchange.

## Comparison to the 2026-05-09 M1 run

Same software, different chip. Hardware delta is the dominant factor;
relative ordering is preserved.

| Framework | M1 p50 (ms) | M5 p50 (ms) | Speedup |
|---|---:|---:|---:|
| sheaf-serve | 32 | 20.5 | 1.6× |
| Raw Ray Serve | 33 | 19.2 | 1.7× |
| BentoML | 7 | 5.6 | 1.3× |

The M5 speedup is real but smaller than its peak single-core delta
versus M1 suggests, because chronos-bolt-tiny inference is the
dominant cost at this load — the substrate overhead, where Ray Serve
and BentoML actually differ, scales less with single-core perf.

## Honest limitations of this run

- **Single RPS point.** A real comparison needs the sweep — see
  [`2026-05-14-sweep-cpu-m5.md`](./2026-05-14-sweep-cpu-m5.md), which
  also surfaces a divergence between sheaf and raw Ray Serve at 100 RPS
  that does not appear here at 50.
- **Single sample of size ~1000.** p99 with N=1000 is stable enough for
  this comparison; the tail at 50 RPS is not noise.
- **Single machine, single replica.** Same caveats as the M1 page:
  Ray Serve's scale-out value cannot be shown on a laptop bench.

## Reproducing this exact run

```bash
# Repo root
uv sync --extra dev --extra time-series
uv pip install bentoml

# Sheaf
.venv/bin/python bench/servers/sheaf_server.py &
sleep 60
.venv/bin/python bench/loadgen.py --url http://127.0.0.1:8000/forecaster/predict \
    --target-rps 50 --duration 20 --warmup 5 --label sheaf
kill %1; .venv/bin/ray stop --force

# Raw Ray Serve
.venv/bin/python bench/servers/ray_serve_server.py &
sleep 60
.venv/bin/python bench/loadgen.py --url http://127.0.0.1:8000/forecaster/predict \
    --target-rps 50 --duration 20 --warmup 5 --label ray-serve
kill %1; .venv/bin/ray stop --force

# BentoML
cd bench/servers
../../.venv/bin/bentoml serve bentoml_server:svc --host 127.0.0.1 --port 8000 &
sleep 10
../../.venv/bin/python ../loadgen.py --url http://127.0.0.1:8000/predict \
    --target-rps 50 --duration 20 --warmup 5 --wire bentoml --label bentoml
kill %1
```

Raw JSON for every run is in [`./2026-05-14-cpu-m5/`](./2026-05-14-cpu-m5/).
